# Задание для IDE-агента: Travel Checklist

## 1. Общая задача

Необходимо разработать простой, быстрый, современный и адаптивный публичный сайт со списками вещей, документов и подготовительных действий для путешествий, командировок, длительных поездок и переездов.

Сайт должен помогать пользователю определить:

* что взять с собой;
* какие документы подготовить;
* что положить в ручную кладь;
* что положить в основной багаж;
* что проверить перед выходом из дома;
* что сделать перед пересечением границы;
* какие вещи потребуются в зависимости от продолжительности поездки;
* какие дополнительные требования могут действовать для выбранной страны;
* что потребуется для командировки, отпуска, учебы или длительного проживания;
* что взять при поездке с ребенком или домашним животным.

Проект должен быть доступен для всех пользователей и бесплатно размещаться через **GitHub Pages**.

Главное требование — создать не одноразовый макет, а полноценный расширяемый стартовый проект, который в будущем можно будет регулярно дополнять.

Должна быть предусмотрена возможность:

* добавлять новые страны;
* добавлять новые регионы;
* добавлять новые типы поездок;
* добавлять новые варианты продолжительности;
* добавлять новые категории;
* добавлять новые пункты чеклиста;
* обновлять рекомендации;
* добавлять новые готовые наборы;
* переводить новые элементы на русский и английский языки;
* развивать проект без полного переписывания интерфейса.

---

## 2. Основные требования

Проект должен соответствовать следующим требованиям:

* полностью статический сайт;
* совместимость с GitHub Pages;
* адаптивная верстка;
* корректная работа на мобильных устройствах;
* русский и английский языки;
* возможность переключения языка;
* дизайн в современной минималистичной стилистике, вдохновленной сайтом Apple;
* преимущественно белый фон;
* черный и темно-серый текст;
* крупная адаптивная типографика;
* большое количество свободного пространства;
* аккуратные карточки;
* плавные анимации;
* понятный конструктор поездки;
* генерация персонального чеклиста;
* сохранение прогресса пользователя;
* отсутствие обязательного backend;
* отсутствие регистрации;
* отсутствие платных API;
* простое дальнейшее наполнение контентом.

Сайт должен восприниматься как готовый публичный продукт для обычных путешественников, а не как административная панель или технический каталог.

---

## 3. Формат проекта и технологический стек

Предпочтительный стек:

* React;
* TypeScript;
* Vite;
* CSS Modules, обычный CSS или легкое решение для стилей;
* JSON или TypeScript-файлы для хранения контента;
* `localStorage` для сохранения данных пользователя;
* GitHub Actions;
* GitHub Pages.

Допускается использование другого легкого frontend-фреймворка, если это не усложняет проект и дальнейшую поддержку.

Не использовать без необходимости:

* backend;
* серверную базу данных;
* Docker для production-развертывания;
* обязательную авторизацию;
* платные сервисы;
* тяжелые UI-фреймворки;
* сложную микросервисную архитектуру;
* зависимости, без которых можно обойтись.

Все основные возможности должны работать непосредственно в браузере пользователя.

---

## 4. Основная концепция сайта

Пользователь открывает сайт и указывает параметры поездки:

1. Страну или регион.
2. Продолжительность поездки.
3. Тип поездки.
4. Сезон или погодные условия.
5. Используемый транспорт.
6. Наличие ручной клади.
7. Наличие основного багажа.
8. Наличие ребенка.
9. Наличие домашнего животного.
10. Необходимость брать рабочую технику.

После выбора параметров сайт формирует персональный чеклист.

Пользователь должен иметь возможность:

* отметить собранные вещи;
* отметить выполненные действия;
* раскрывать и сворачивать категории;
* добавлять собственные пункты;
* удалять собственные пункты;
* скрывать выполненные пункты;
* сбрасывать прогресс;
* копировать список;
* распечатывать список;
* делиться конфигурацией поездки через ссылку.

Прогресс должен сохраняться в `localStorage`.

После обновления страницы пользователь не должен терять:

* выбранную страну;
* продолжительность поездки;
* тип поездки;
* дополнительные параметры;
* собственные пункты;
* отмеченные позиции;
* выбранный язык.

## 5. Языки сайта

Сайт должен полностью поддерживать два языка:

* русский — `ru`;
* английский — `en`.

Перевод должен охватывать весь интерфейс и весь контент.

Не допускается ситуация, когда часть интерфейса остается на русском, а часть отображается на английском языке.

Исключения:

* названия брендов;
* названия сервисов;
* международные аббревиатуры;
* общепринятые названия технологий.

---

## 6. Переключение языка

В верхней навигации разместить переключатель языка.

Допустимые варианты:

```text
RU | EN
```

или:

```text
Русский
English
```

Требования:

* язык переключается без потери данных пользователя;
* выбранный язык сохраняется в `localStorage`;
* при первом открытии проверяется язык браузера;
* при `navigator.language`, начинающемся с `ru`, выбирается русский;
* для остальных языков по умолчанию используется английский;
* пользователь всегда может вручную изменить язык;
* при переключении языка не должен сбрасываться чеклист;
* язык должен влиять на SEO-метаданные и атрибут `lang`.

## 10. Локализация дат, чисел и множественного числа

Использовать стандартные браузерные API:

* `Intl.DateTimeFormat`;
* `Intl.NumberFormat`;
* `Intl.ListFormat`;
* `Intl.PluralRules`;
* `Intl.DisplayNames`.

Не собирать вручную строки, зависящие от количества.

Необходимо корректно поддержать:

```text
1 пункт
2 пункта
5 пунктов
```

и:

```text
1 item
2 items
```

Английский текст должен быть естественным, а не дословным машинным переводом.

Например, не использовать:

```text
Collect checklist
```

Использовать:

```text
Create a checklist
```

Для вещей использовать:

```text
Packed
```

Для выполненных действий:

```text
Completed
```

## 11. Разделение поездок по продолжительности

Необходимо создать отдельные сценарии поездок.

### 11.1. Поездка на один день

Продолжительность:

* один день;
* без ночевки.

Примеры:

* поездка в соседний город;
* короткая командировка;
* экскурсия;
* мероприятие;
* прогулка.

Основные пункты:

* паспорт;
* телефон;
* зарядное устройство;
* пауэрбанк;
* деньги;
* банковская карта;
* вода;
* лекарства первой необходимости;
* салфетки;
* одежда по погоде;
* билет;
* адрес назначения;
* обратный маршрут.

### 11.2. Поездка на выходные

Продолжительность:

* 2–3 дня.

Дополнительно учитывать:

* нижнее белье;
* носки;

* бронирование жилья;
* билеты;
* зарядные устройства.

### 11.3. Короткий отпуск

Продолжительность:

* 4–7 дней.

* сменную обувь;
* расширенную аптечку;
* косметику;
* средства гигиены;

### 11.4. Поездка на две недели

Продолжительность:

Дополнительно учитывать:

* адаптеры для розеток;

### 11.5. Длительная поездка

Продолжительность:

* 15–30 дней.

Дополнительно учитывать:

* лекарства на весь период;
* запасные очки;
* запасные линзы;
* документы для длительного пребывания;
* медицинскую страховку;

Продолжительность:

На сайте обязательно разместить заметное предупреждение.

Русская версия:

> Правила въезда, визовые требования, требования к документам, ограничения на провоз вещей и правила использования лекарственных препаратов могут изменяться. Перед поездкой проверяйте актуальную информацию на официальных сайтах государственных органов, консульств, авиакомпаний и перевозчиков.

Английская версия:

> Entry rules, visa requirements, document requirements, baggage restrictions and medication regulations may change. Always verify current information through official government, embassy, airline and transport provider websites before travelling.

Для страновых рекомендаций предусмотреть поля:

```json
{
  "sourceUrl": "",
  "lastReviewed": "2026-06-01"
}
```

Не отображать непроверенную информацию как гарантированно актуальную.
